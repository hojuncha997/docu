웹서버

쉡서버는 서버 소프트웨어, 또는 이 소프트웨어를 굳공하는 데 사용되는 하드웨어를 지칭한다.
웹 서버는 월드와이드웹에서의 클라이언트의 리퀘스트를 만족시키는 역할을 한다. 하나의 웹서버는
일반적으로 하나 또는 그 이상의 웹사이트를 가질 수 있으며, HTTP 또는 다른 여러 프로토콜을 통한
네트워크 리퀘스트를 처리한다.

웹서버의 주된 기능은 웹페이지를 저장했다가 클라이언트에게 전달하는 것이다. 클라이언트와 서버 사이으이 소통은
HTTP를 통해 가능하다. 클라이언트에게 가장 빈번히 전달되는 페이지는 HTML 문서이다. 이 문서들은 텍스트 외에도
이미지, Style sheet, 스크립트를 포함할 수 있다.

웹브라우저 또는 웹크롤러로 대변도는 user agent는 HTTP를 사용해서 특정 리소스를 서버에게 요청(request)함으로써
소통을 시작한다. 서버는 리소스에 대한 내용을 전달함으로써 응답(respond)한다. 만약 그것이 불가능하다면 에러메시지로
응답한다. 이 리소스는 서버의 저장장치에 저장된 실제의 파일이다. 그러나 이러한 경우가 반드시 필연적인 것은 아니며,
웹서버가 구동되는 데 반드시 동반되어야 하는 것은 아니다.

웹서버의 주기능은 content를 제공하는 것이다. 그러나 완전한 HTTP의 구현은 클라이언트로부터 content를 수신하는
방식도 포함한다. 이러한 기능은 클라이언트의 파일업로드, 양식 제출 등을 위해 사용된다.

상당량의 일반적인 웹서버들이 ASP, PHP 등의 서버사이드 스크립팅을 지원한다. 이것은 웹서버의 행위(또는 조작)이
개별 파일에도 적용될 수 있다는 뜻이다. 실제 서버 소프트웨어는 변화되지 않았음에도 말이다. 일반적으로 이러한 기능은
정적인 문서를 반환하기 보다는 역동적인 HTML 문서를 생성하기 위해 사용된다. 이러한 동적인 문서들은 DB의 
정보를 조회하거나 조작하는 데 이용된다. 정적인 문서들은 훨씬 처리하기 빠르지만 역동적인 content를 제공하지는 못한다.

웹서버는 프린터, 라우터, 공유기, 웹캠 등에서도 찾을 수 있으며, 오직 local 네트워크에서 사용될 수도 있다.
웹서버는 여러 장치들을 통제하고 그것들을 질의를 모니터링 하는 시스템의 일부로 사용될 수도 있다. 이것은 일반적으
웹브라우저 말고는 클라이언트 컴퓨터에 추가적인 소프트웨어를 설치하지 않아도 된다는 것을 의미한다.

---

웹서버의 구분

1) 하드웨어
	: web server가 설치되어 있는 컴퓨터
2) 소프트웨어
	: 웹브라우저 클라이언트로부터 HTTP 요청을 받아 정적인 컨텐츠(.html, .jpeg, .css 등)을
	제공하는 컴퓨터 프로그램
	
웹서버의 기능
	- HTTP 프로토콜을 기반으로 하여 클라이언트(웹 브라우저 또는 웹크롤러)의 요청을 서비스하는 기능 담당
	- 요청에 따라 아래의 두 가지 기능 중 적절하게 선택하여 수행한다.
	
	기능1) 
		- 정적인 컨텐츠 제공
		- WAS를 거치지 않고 바로 자원을 제공
		  
	기능2)
		- 동적인 컨텐츠 제공을 위한 요청 전달
		- 클라이언트의 요청(request)를 WAS에 보내고, WAS가 처리한 결과를 클라이언트에게
			전달(응답, reponse)한다.
		- 클라이언트는 일반적으로 웹브라우저를 의미한다.
		
	웹서버의 예)
		- 아파치 서버, IIS
